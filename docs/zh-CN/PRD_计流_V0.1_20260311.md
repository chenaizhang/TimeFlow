# 《计流》PRD（V0.1 精简版）

- 版本：V0.1
- 日期：2026-03-11
- 平台：Flutter（iOS / Android）
- 一句话定义：一款面向个人用户的「待办计时 + 时间分配分析」工具。

## 1. 版本目标

V0.1 只做一个闭环：

1. 用户可创建待办集与待办，发起正向计时或倒计时。
2. 结束计时后自动沉淀专注记录（小于 1 分钟可放弃保存）。
3. 用户可在统计页按日/周/月/自定义区间查看时间分布。
4. 用户可查看专注记录、导出备份、导入备份、分享统计海报。

## 2. 版本范围

### 2.1 In Scope（V0.1）

- 待办集管理：新增、编辑、删除。
- 待办管理：新增、编辑、删除；同组重名校验。
- 计时模式：正向计时、倒计时（30min / 1h / 自定义，最长 5h）。
- 倒计时完成提醒：可配置震动、铃声。
- 运行中计时页：显示计时器与待办名，禁止返回退出。
- 记录生成：停止时写入记录；<1 分钟弹窗确认并默认不计入统计。
- 统计页：累计专注、当日专注、专注时长分布。
- 分布维度：日 / 周 / 月 / 自定义。
- 饼图交互：点击高亮、点击空白恢复、拖动旋转。
- 历史记录页：按日期查看记录；日历支持左右滑月与年月快速定位。
- 数据能力：本地 SQLite 存储；备份导出/导入 JSON。
- 分享能力：统计海报预览、保存到相册、系统分享。

### 2.2 Out of Scope（V0.1 不做）

- 云同步与多设备实时合并。
- 团队协作、社交体系。
- 番茄钟复杂流程（如多阶段循环）。
- AI 复盘建议。
- 导入番茄todo数据。

## 3. 详细需求

### 3.1 待办计时模块

#### 3.1.1 待办集

- 待办集名必填，长度 1~20。
- 待办集名不可重复。
- 允许删到 0 个待办集。
- 删除待办集时：
  - 若其下待办有历史记录，待办与待办集逻辑删除，历史记录保留。
  - 若无历史记录，可物理删除。

#### 3.1.2 待办

- 字段：名称、所属待办集、计时模式、倒计时秒数、震动开关、铃声开关。
- 名称必填、长度 1~20。
- 同一待办集下名称不可重复（同名视为同一个）。
- 颜色由系统自动分配，用户不手动指定。
- 删除规则与待办集一致：保留历史记录完整性。
- 若历史存在且删除后重建同名待办，删除期间原颜色不可被复用（避免历史颜色冲突）。

#### 3.1.3 计时流程

- 同一时刻仅允许一个进行中的计时。
- 点击「开始」后有转场动效进入计时页。
- 计时页展示：
  - 计时器（正向增长/倒计时递减）
  - 待办名称（位于计时器下方）
  - 状态文案（正向计时中 / 倒计时中）
  - 结束按钮（暂停预留到后续版本）
- 结束逻辑：
  - > = 1 分钟：保存记录并刷新统计。
  - < 1 分钟：弹窗确认是否结束；结束则不入统计。
- App 退后台/恢复或重启后：从 `current_timer` 恢复进行中计时状态。

### 3.2 统计模块

#### 3.2.1 累计专注概览

- 展示：累计次数、累计时长、日均时长。
- 大数适配：数字不能被截断。
- 文案规则：当小时数 >= 1000 时，总计时长忽略分钟，仅显示小时。

#### 3.2.2 当日专注

- 展示：所选日期的专注次数、专注时长。
- 日期默认今天。
- 支持前后切换与滚轮选日（范围：2026-01-01 至今天）。

#### 3.2.3 专注时长分布

- 展示：范围日期、分段按钮（日/周/月/自定义）、饼图、总计、日均、下方待办明细。
- 日期文案：
  - 单日仅显示 `yyyy-MM-dd`。
  - 多日显示 `start ~ end`。
- 饼图规则：
  - 颜色与计时页待办颜色一致。
  - 不显示百分比，显示待办与时长。
  - 标签引导线固定为「先斜后横」。
  - 时长文本位于线末端，且不与线/饼图重叠。
  - 仅选中某扇区时外拉；点空白恢复完整圆。
  - 支持拖动旋转，旋转时页面禁止滚动。

#### 3.2.4 专注记录

- 支持按天查看明细列表（待办名、开始-结束时间、时长）。
- 日历中有记录的日期以圈标记。
- 日历支持左右滑动切月并带动画。
- 支持年月滚轮快速定位。

### 3.3 分享与备份

#### 3.3.1 分享海报

- 从统计页进入居中弹窗预览（非底部抽屉）。
- 海报包含：日期、当日专注模块、专注时长分布模块、明细列表。
- 模块结构与统计页一致，仅缩小尺寸。
- 导出图比例接近 1:1；避免底部大面积空白；饼图保持正圆不被压扁。

#### 3.3.2 数据备份

- 统计页右上角「...」菜单支持：
  - 导出备份（JSON）
  - 导入备份（JSON）
- 导入前二次确认，导入覆盖本地数据。

## 4. 数据设计（SQLite）

### 4.1 `project_groups`（待办集）

- `id`, `name`, `sort_order`, `is_deleted`, `created_at`, `updated_at`

### 4.2 `projects`（待办）

- `id`, `name`, `group_id`, `timer_mode`, `countdown_seconds`
- `color_value`, `enable_vibration`, `enable_ringtone`
- `is_enabled`, `is_deleted`, `sort_order`, `created_at`, `updated_at`

### 4.3 `focus_sessions`（专注记录）

- `id`, `project_id`, `start_time`, `end_time`, `duration_seconds`
- `status`, `record_date`, `created_at`, `updated_at`

### 4.4 `current_timer`（进行中计时）

- `id=1`, `project_id`, `start_time`, `status`, `last_sync_time`
- `timer_mode`, `target_seconds`

## 5. 统计口径

- 有效记录：`duration_seconds >= 60` 且 `status = completed`。
- 时间归属：按开始时间所在日期归属。
- 累计日均：累计总时长 / 有记录天数。
- 区间日均：区间总时长 / 区间自然天数。

## 6. 验收清单（V0.1）

3. 倒计时支持 30min/1h/自定义（<=5h）+ 震动/铃声开关。
4. <1 分钟结束时弹窗确认，不纳入统计。
5. 累计专注、当日专注、分布统计计算正确。

7. 饼图交互（选中/恢复/旋转）可用且标签可读。
8. 专注记录页支持日历切换、滑动切月、年月快速定位。
9. 支持导出备份、导入备份。

# 《计流》PRD（V0.2）

- 版本：V0.2
- 日期：2026-03-12
- 平台：Flutter（iOS / Android）
- 上一版本：V0.1（已完成）
- 一句话定义：在 V0.1「待办计时 + 统计分析」闭环基础上，补齐复盘、暂停与月/年趋势分析能力。

## 1. 版本目标

V0.2 聚焦三件事：

1. 修复关键体验问题（安卓提醒稳定性、待办集展开黑边）。
2. 完成「计时结束复盘 + 记录详情可操作」闭环。
3. 升级统计页（本月时段分布、月度趋势、年度趋势）。

## 2. 版本范围

### 2.1 In Scope（V0.2）

- Bug 修复：
  - 安卓端铃声与震动不可用/不稳定。
  - 待办集展开时上下边缘黑边。
- 新功能：
  - 专注记录详情弹窗（含心得展示/补填入口）与删除。
  - 每次有效计时结束后的心得弹窗（填写并保存/稍后再填）。
  - 计时页暂停机制（单次计时总暂停额度 180 秒）。
  - 本月专注时段分布柱状图（按小时分桶）。
  - 月度专注统计折线图（按日）。
  - 年度专注统计折线图（按月）。

### 2.2 Out of Scope（V0.2 不做）

- 云同步与多设备冲突合并。
- 团队协作、社交功能。
- 第三方平台数据导入。

## 3. Bug 修复需求

### 3.1 安卓端铃声与震动不可用

#### 目标行为

- 倒计时/暂停结束时：
  - `铃声=开`：播放系统提示音。
  - `震动=开`：触发一次清晰震动反馈。
- 应用不在前台时：
  - 发送系统通知提醒（如“暂停结束”）。

#### 验收标准

1. 安卓真机可稳定触发铃声与震动。
2. 后台或锁屏状态下可通过通知触达用户。
3. 对应开关关闭后，不触发对应提醒。

### 3.2 待办集展开黑边

#### 目标行为

- 展开/收起状态下卡片四边视觉一致，无上下黑边。
- 阴影、裁剪、圆角在四边表现一致。

#### 验收标准

1. iOS/Android 均无黑边。
2. 列表滚动与展开动画期间无闪烁/抖动。

## 4. 新功能需求

### 4.1 专注记录详情与删除

#### 需求说明

- 在「专注记录页」点击任意记录，弹出详情弹窗。
- 弹窗展示：
  - 待办名称
  - 日期
  - 开始时间
  - 结束时间
  - 持续时长
  - 心得（可为空）
- 弹窗提供「删除记录」按钮，并二次确认。

#### 交互规则

- 删除后：
  - 记录从列表消失。
  - 统计页同步刷新。
- 无心得时展示 `点击填写心得`，点击可继续补填。

#### 验收标准

1. 点击记录可稳定弹出详情。
2. 删除后记录与统计同步更新。
3. 有/无心得状态展示正确。

### 4.2 计时结束心得弹窗

#### 需求说明

- 每次有效计时结束后，弹出心得弹窗。
- 提供两个主要动作：
  - `填写并保存`
  - `稍后再填`

#### 字段规则

- 心得可为空文本，长度上限 500 字。
- 选择 `稍后再填` 时：
  - 记录先保存（不阻断主流程）。
  - 标记为待补心得状态。

#### 补填入口

- 在专注记录详情弹窗支持补填/更新心得。

#### 验收标准

1. 每次有效计时结束后必弹。
2. 稍后再填不丢记录。
3. 补填后可在详情中查看最新心得。

### 4.3 计时页暂停机制（V0.2 关键）

#### 需求说明

- 单次计时总暂停额度为 180 秒。
- 暂停期间计时停止增长/递减。
- 暂停弹窗需提示：为避免打断专注，单次计时最多暂停 3 分钟。

#### 详细规则

1. 单次计时可多次暂停，但累计不超过 180 秒。
2. 支持提前结束当前暂停。
3. 已消耗暂停时长实时扣减；剩余额度可继续使用。
4. 剩余额度为 0 时：
  - 本次计时不可再暂停。
  - 暂停按钮禁用且有明确文案。
5. 暂停结束时：
  - 前台：震动一次。
  - 后台：震动并发送系统通知提醒。
6. 暂停期间不能因误触导致状态异常。

#### 时长口径

- 统计时长不包含暂停时长。

#### 验收标准

1. 单次计时暂停总时长严格上限 180 秒。
2. 提前结束后额度扣减正确。
3. 额度耗尽后暂停不可用。
4. 前台/后台结束提醒符合要求。

### 4.4 本月专注时段分布（按小时柱状）

#### 需求说明

- 新增「本月专注时段分布」模块。
- 统计口径：
  - 以小时分桶（0~23 共 24 桶）。
  - 每桶为该月对应小时累计专注时长。
  - 以 4 点为锚点裁剪两端连续空白桶（优先裁去典型睡眠空白区间）。
- 图表形式：竖向柱状图。

#### 交互规则

- 标题右侧支持月份切换与选择（年+月滚轮）。
- 横轴标签显示 3 的倍数小时（0/3/6/...）。
- 柱顶显示小时值（小字号、低对比）。

#### 验收标准

1. 切月后图表数据正确刷新。
2. 裁剪逻辑与 4 点锚点规则一致。
3. 无数据月份展示空状态文案。

### 4.5 月度专注统计（按日折线）

#### 需求说明

- 新增「月度专注统计」模块。
- 统计口径：
  - x 轴：当月每日。
  - y 轴：当天专注小时数（可保留 1 位小数）。
- 图表形式：平滑折线（拟合曲线），且不产生负值。

#### 交互规则

- 支持月份切换与选择（年+月滚轮）。
- 点位显示圆点与具体数值。
- 长按显示横/竖辅助线。
- 单击点位或相邻区间后约 0.2 秒跳转到专注记录页对应日期。

#### 验收标准

1. 每日小时数与记录汇总一致。
2. 跨月切换流畅、无跳跃。
3. 点位查看与跳转逻辑正确。

### 4.6 年度专注统计（按月折线）

#### 需求说明

- 新增「年度专注统计」模块。
- 统计口径：
  - x 轴：当年每月（1~12）。
  - y 轴：当月专注小时数。
- 图表形式：平滑折线。

#### 交互规则

- 支持年份切换与选择（仅年份滚轮）。
- 交互风格复用月度专注统计（点位、辅助线）。
- 单击点位后约 0.2 秒跳转到专注记录页该月第一日。

#### 验收标准

1. 年内每月值映射正确。
2. 切年后数据与坐标同步更新。
3. 点位跳转到对应月份首日记录页。

## 5. 数据设计与统计查询（SQLite）

### 5.1 `focus_sessions`

- `note TEXT NULL`：心得文本。
- `note_pending INTEGER NOT NULL DEFAULT 0`：是否待补心得（0/1）。

### 5.2 `current_timer`

- `status TEXT`：计时状态（运行/暂停）。
- `paused_seconds_total INTEGER NOT NULL DEFAULT 0`：当前计时已消耗暂停秒数。
- `pause_started_at TEXT NULL`：暂停开始时间。

### 5.3 统计查询新增

- 本月时段分布：按小时（0~23）聚合。
- 月度折线：按 `record_date` 按日聚合。
- 年度折线：按 `record_date` 按月聚合。

## 6. 统一交互与边界规则

- 全部时间切换窗口最早可选日期统一为 `2017-01-01`，最晚为“今天”。
- 图表切换月份/年份时，页面不应抖动或跳跃。
- 删除记录后，若当前日期无数据，列表应正确回到空状态。
- 暂停中强杀 App，恢复后应按真实时间恢复暂停额度与状态。

## 7. 非功能要求

- 图表相邻月份/年份切换响应目标：< 300ms。
- 单月 2,000+ 条记录时，统计页滚动与交互保持可用。
- 安卓提醒稳定率目标：100 次触发成功率 >= 99%。

## 8. 验收清单（V0.2）

1. 安卓提醒（铃声/震动/后台通知）符合开关与前后台预期。
2. 待办集展开黑边问题修复，滚动与动画稳定。
3. 专注记录详情可查看、删除、补填心得，并实时反映到统计页。
4. 每次有效计时结束都会出现心得弹窗，支持填写/稍后再填。
5. 暂停机制满足 180 秒总额度、可提前结束、额度耗尽禁用、时长口径正确。
6. 本月专注时段分布图的聚合、裁剪、切月交互正确。
7. 月度专注统计的曲线、点位、辅助线、跳转逻辑正确。
8. 年度专注统计的按月聚合、切年交互、跳转逻辑正确。
9. 数据迁移后老数据可读取，页面无崩溃。

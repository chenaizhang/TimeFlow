# 《计流》PRD（V0.2）

- 版本：V0.2
- 日期：2026-03-12
- 平台：Flutter（iOS / Android）
- 上一版本：V0.1（已完成）
- 一句话定义：在 V0.1「记录 + 统计」闭环基础上，补齐暂停机制、记录编辑与月/年趋势统计。

## 1. 版本目标

V0.2 聚焦三件事：

1. 解决影响体验的关键问题（安卓提醒失效、代办集黑边）。
2. 完成「计时结束复盘」与「专注记录可编辑」能力闭环。
3. 增强统计页：新增月度时段分布、月度趋势、年度趋势。

## 2. 版本范围

### 2.1 In Scope（V0.2）

- Bug 修复：
  - 安卓端铃声与震动不可用。
  - 代办集展开时上下轻微黑边。
- 新功能：
  - 专注记录详情弹窗（含心得）与删除。
  - 每次计时结束后的心得弹窗（可填写/稍后再填）。
  - 计时页暂停机制（单次计时总暂停额度 3 分钟）。
  - 月度专注时段分布图（按 24 小时分桶）。
  - 月度专注统计折线图（按日）。
  - 年度专注统计折线图（按日）。

### 2.2 Out of Scope（V0.2 不做）

- 云同步、多设备冲突合并。
- 任务协作与社交。
- 导入第三方平台数据。

## 3. Bug 修复需求

### 3.1 安卓端铃声与震动不可用

#### 问题描述

- 在安卓设备上，倒计时结束后铃声或震动未触发，或触发不稳定。

#### 目标行为

- 当倒计时结束：
  - 若「铃声=开」，播放系统提示音。
  - 若「震动=开」，触发一次清晰震动反馈。
- 若应用不在前台：
  - 通过系统通知提醒用户「暂停结束」。

#### 验收标准

1. 安卓真机可稳定触发铃声与震动。
2. 在后台或锁屏后，结束提醒可通过通知触达。
3. 关闭对应开关后不触发相应提醒。

### 3.2 代办集展开黑边

#### 问题描述

- 代办集展开时卡片上下边缘出现轻微黑边，与左右边视觉不一致。

#### 目标行为

- 展开/收起状态下卡片边缘一致，不出现上下黑边。
- 阴影、裁剪、圆角在四边表现一致。

#### 验收标准

1. iOS/Android 均无黑边。
2. 列表滚动与展开动画期间视觉稳定。

## 4. 新功能需求

### 4.1 专注记录详情与删除

#### 需求说明

- 在「专注记录页」点击任意记录，弹出详情弹窗。
- 弹窗展示：
  - 待办名称
  - 日期
  - 开始时间
  - 结束时间
  - 持续时长
  - 心得（可为空）
- 弹窗底部提供「删除记录」按钮，并二次确认。

#### 交互规则

- 删除后：
  - 记录从列表消失。
  - 统计页数据实时刷新。
- 心得只读展示，编辑入口放在「稍后补填心得」流程（见 4.2）。

#### 验收标准

1. 点击记录可稳定弹出详情。
2. 删除后记录与统计同步更新。
3. 无心得时显示 `点击填写心得`。

### 4.2 计时结束心得弹窗

#### 需求说明

- 每次计时结束后，弹出心得填写弹窗。
- 提供两个主要动作：
  - `填写并保存`
  - `稍后再填`

#### 字段规则

- 心得：可为空文本，建议长度 0~500 字。
- 选择 `稍后再填` 时：
  - 记录先保存（不阻断主流程）。
  - 标记为「待补心得」。

#### 补填入口

- 在专注记录详情弹窗中支持补填/更新心得。

#### 验收标准

1. 每次有效计时结束后必弹。
2. 选择稍后再填不丢记录。
3. 补填后可在详情中查看最新心得。

### 4.3 计时页暂停机制（V0.2 关键）

#### 需求说明

- 一次计时总暂停额度为 3 分钟（180 秒）。
- 暂停时，计时器停止增长/递减。
- 暂停弹窗显示剩余暂停倒计时，并提示：
  - 「为避免暂停过长时间打断专注，单次计时最多暂停 3 分钟」。

#### 详细规则

1. 单次计时可多次暂停，但累计不超过 180 秒。
2. 可提前结束当前暂停。
3. 当前暂停已消耗时间会从总额度扣除；剩余额度保留给本次计时后续使用。
4. 当剩余额度为 0：
   - 本次计时不允许再次暂停。
   - 暂停按钮禁用并有明确文案。
5. 暂停结束时：
   - 震动一次。
   - 若应用不在前台，发送系统通知提醒。
6. 暂停期间用户不能误触关闭关键弹窗导致状态异常。

#### 时长口径

- 计时统计时长不包含暂停消耗时间。

#### 验收标准

1. 单次计时暂停总时长上限严格为 180 秒。
2. 提前结束后额度正确扣减并可继续使用剩余额度。
3. 额度用尽后暂停不可用。
4. 暂停结束在前台有震动，在后台有通知提醒。

### 4.4 月度专注时长分布（按时段）

#### 需求说明

- 新增「本月专注时段分布」图表模块。
- 统计口径：
  - 按小时分桶（0~23 共 24 桶）。
  - 每桶为该月对应时段累计专注时长。
- 图表形式：竖向柱状图（每小时一根）。

#### 交互

- 模块标题右侧支持切换月份（交互样式参考当日专注切换）。
- 显示当前月份（`yyyy-MM`）。

#### 验收标准

1. 切月后图表数据正确刷新。
2. 24 个时段完整展示。
3. 无数据月份展示空状态与引导文案。

### 4.5 月度专注统计（按日折线）

#### 需求说明

- 新增「月度专注统计」模块。
- 统计口径：
  - x 轴：当月每日。
  - y 轴：当天专注小时数（可保留 1 位小数）。
- 图表形式：折线图。

#### 交互

- 支持月份切换（交互样式参考当日专注）。
- 支持查看点位具体值（tooltip）。

#### 验收标准

1. 每日小时数与记录汇总一致。
2. 跨月切换流畅，数据正确。

### 4.6 年度专注统计（按日折线）

#### 需求说明

- 新增「年度专注统计」模块。
- 统计口径：
  - x 轴：当年每一天（1~365/366）。
  - y 轴：当日专注小时数。
- 图表形式：折线图。

#### 交互

- 支持年份切换（交互样式参考当日专注）。
- 支持点位查看具体日期与时长。

#### 验收标准

1. 年内每日值可正确映射。
2. 闰年天数处理正确。
3. 大数据量下图表滚动与渲染可接受。

## 5. 数据设计变更（SQLite）

### 5.1 `focus_sessions` 新增字段

- `reflection_note TEXT NULL`：计时心得。
- `reflection_status TEXT NOT NULL DEFAULT 'empty'`：`empty` / `pending` / `filled`。
- `reflection_updated_at TEXT NULL`：心得最后更新时间。

### 5.2 `current_timer` 新增字段

- `timer_status TEXT NOT NULL DEFAULT 'running'`：`running` / `paused`。
- `pause_budget_seconds INTEGER NOT NULL DEFAULT 180`：本次计时总暂停额度。
- `pause_used_seconds INTEGER NOT NULL DEFAULT 0`：本次计时已使用暂停时长。
- `pause_started_at TEXT NULL`：本次暂停开始时间。

### 5.3 统计查询新增

- 月度时段分布：按开始时间小时（0~23）聚合。
- 月度折线：按 `record_date` 按日聚合。
- 年度折线：按 `record_date` 全年按日聚合。

## 6. 交互与异常场景

- 计时结束心得弹窗被中断（来电/切后台）后，恢复前台应继续可见或可再次进入补填流程。
- 暂停中强杀 App：恢复时应按真实时间扣减暂停额度，并更新状态。
- 记录删除后，若当前筛选日期无记录，列表正确回空状态。
- 图表切换月份/年份时，页面不应抖动或跳跃。

## 7. 非功能要求

- 图表切换（同一维度相邻月份/年份）响应时间目标 < 300ms。
- 单月 2,000+ 条记录时，统计页仍可流畅滚动与交互。
- 安卓提醒稳定率：100 次触发中成功率 >= 99%。

1. 安卓端铃声与震动在前台/后台均符合预期。
2. 代办集展开无上下黑边。
